# Experiment-Navigation

This repository contains the offline navigation-planning and dataset-generation pipeline used for collecting local motion samples for a quadruped robot navigation experiment. The planner generates collision-free SE(2) trajectories using an RRT-based planner, and the resulting trajectories are converted into supervised samples for training a diffusion-policy-based local motion generator.

## Planner Overview

The RRT-based planner is implemented in:

```text
go2_planner_offline/planner.cc
```

The planner operates in an SE(2) configuration space:

```text
q = (x, y, theta)
```

where `x` and `y` represent the planar position of the robot, and `theta` represents the robot heading. The robot is modelled as a rectangular footprint, and obstacles are modelled as rectangular objects with an additional inflated safety margin for collision checking.

For each planning run, the RRT planner generates a feasible path from the start pose toward the goal region. The goal is considered reached when the final waypoint lies within the specified goal-tolerance radius.

For example, suppose the RRT planner generates the following path:

```text
p1, p2, p3, p4, p5
```

Here:

```text
p1 = start pose
p5 = final RRT waypoint inside the goal-tolerance region
```

The final waypoint `p5` does not necessarily coincide exactly with the goal pose. It only needs to lie within the goal-tolerance region.

## Navigation Dataset Format

The generated dataset is stored inside the `Navigation_Dataset/` directory. Each planning run is indexed by `i`, where `i` is the run index.

### 1. `waypoints/`

This folder stores the complete SE(2) waypoint trajectory generated by the RRT planner.

Example file:

```text
Navigation_Dataset/waypoints/se2_waypoints_i.txt
```

If the generated path is:

```text
p1, p2, p3, p4, p5
```

then `se2_waypoints_i.txt` stores all waypoints:

```text
p1
p2
p3
p4
p5
```

Each row has the format:

```text
x, y, theta
```

### 2. `Present_Robot_Pose/`

This folder stores the current robot pose used as the input state for each local motion-learning sample.

Example file:

```text
Navigation_Dataset/Present_Robot_Pose/curr_pose_i.txt
```

For the path:

```text
p1, p2, p3, p4, p5
```

the current-pose file stores:

```text
p1
p2
p3
p4
```

The final waypoint `p5` is not included as a current pose because the dataset is designed to learn local planner edges:

```text
p1 -> p2
p2 -> p3
p3 -> p4
p4 -> p5
```

### 3. `Target_Pose/`

This folder stores the target pose corresponding to each current robot pose.

Example file:

```text
Navigation_Dataset/Target_Pose/local_target_pose_i.txt
```

For the path:

```text
p1, p2, p3, p4, p5
```

the target-pose file stores:

```text
p2
p3
p4
p5
```

Therefore, each row in `Present_Robot_Pose/` is paired with the corresponding row in `Target_Pose/`:

```text
current pose: p1  -> target pose: p2
current pose: p2  -> target pose: p3
current pose: p3  -> target pose: p4
current pose: p4  -> target pose: p5
```

This creates a dataset for learning local feasible planner transitions.

### 4. `Local_Obstacle_Info/`

This folder stores obstacle information expressed in the robot's local coordinate frame.

Example file:

```text
Navigation_Dataset/Local_Obstacle_Info/local_obs_i.txt
```

For each current robot pose, the obstacle centres are transformed from the world frame into the robot frame. Each row corresponds to one current pose in `Present_Robot_Pose/`.

For example, if the current pose is `p_k`, each obstacle is represented as:

```text
[obs_rel_x, obs_rel_y, obs_rel_dist]
```

where:

```text
obs_rel_x    = obstacle x-position in the robot frame
obs_rel_y    = obstacle y-position in the robot frame
obs_rel_dist = Euclidean distance from the robot pose to the obstacle centre
```

For the path:

```text
p1, p2, p3, p4, p5
```

the obstacle file stores obstacle information relative to:

```text
p1
p2
p3
p4
```

The final waypoint `p5` is not included because it is not used as a current pose in the planner-edge imitation dataset.

### 5. `Local_Goal_Info/`

This folder stores the relative goal position with respect to each current robot pose.

Example file:

```text
Navigation_Dataset/Local_Goal_Info/local_goal_info_i.txt
```

Each row has the format:

```text
goal_rel_x, goal_rel_y, goal_rel_dist
```

where:

```text
goal_rel_x    = goal x-position in the robot frame
goal_rel_y    = goal y-position in the robot frame
goal_rel_dist = Euclidean distance from the current robot pose to the goal
```

Although the current diffusion-policy input primarily uses the current pose, target pose, and local obstacle information, the local goal information is also stored for analysis and possible future goal-conditioned extensions.

### 6. `controls/`

This folder stores velocity commands generated from consecutive RRT waypoints.

Example file:

```text
Navigation_Dataset/controls/controls_i.txt
```

Each row corresponds to one local transition:

```text
p1 -> p2
p2 -> p3
p3 -> p4
p4 -> p5
```

Each command has the format:

```text
vx, vy, vtheta
```

where:

```text
vx     = forward velocity
vy     = lateral velocity
vtheta = angular velocity
```

### 7. `planning_tree/`

This folder stores the RRT search tree generated during planning.

Example file:

```text
Navigation_Dataset/planning_tree/rrt_tree_i.txt
```

This file is mainly used for visualization and debugging of the RRT search process.

### 8. `planner_logs/`

This folder stores terminal logs for each planning run.

Example file:

```text
Navigation_Dataset/planner_logs/planner_run_i.log
```

The log file records useful information such as planning success, best path cost, selected seed, planning time, and file-save locations.

### 9. `Planned_Path_Plot/`

This folder stores visualization outputs generated by:

```text
go2_planner_offline/visualize.py
```

Example file:

```text
Navigation_Dataset/Planned_Path_Plot/planned_path_plot_i.png
```

The visualization includes:

* the SE(2) planning environment,
* the rectangular robot footprint,
* rectangular obstacles,
* inflated obstacle safety margins,
* the RRT search tree,
* the final planned path,
* local obstacle-distance lines,
* local goal-distance lines,
* and the goal-tolerance region shown as a green circle.

### 10. `Timestep/`

This folder stores the planning time for each run.

Example file:

```text
Navigation_Dataset/Timestep/plan_time_i.txt
```

The stored value represents the time taken by the winning RRT run to find the best goal-reaching path.

### 11. `query/`

This folder stores a copy of the `query.cfg` file used for each planning run.

Example file:

```text
Navigation_Dataset/query/query_i.cfg
```

This is important because `query.cfg` contains the dynamic planning problem definition, including:

* start pose,
* goal pose,
* obstacle positions,
* obstacle sizes,
* obstacle orientations,
* and other problem-specific parameters.

Saving one query file per run ensures that each dataset sample can be traced back to the exact planning environment that generated it.

## Learning Objective

The dataset is designed for learning local feasible planner transitions. Given a current robot pose, a local target pose selected from the RRT trajectory, and local obstacle information expressed in the robot frame, the diffusion policy learns to generate a feasible local motion distribution similar to the RRT planner transition.

For a trajectory:

```text
p1, p2, p3, p4, p5
```

the learning samples are:

```text
(p1, p2, local_obstacle_info_at_p1)
(p2, p3, local_obstacle_info_at_p2)
(p3, p4, local_obstacle_info_at_p3)
(p4, p5, local_obstacle_info_at_p4)
```

The objective is to train the diffusion policy to imitate feasible local RRT edges while conditioning on local geometric observations.

## Summary

This dataset supports planner-guided diffusion policy learning for local navigation. The RRT planner provides collision-free local transitions, and the generated dataset converts these transitions into supervised learning samples. The policy is expected to learn a distribution over feasible local motions conditioned on:

```text
current robot pose,
target pose,
local obstacle information.
```

This formulation allows the diffusion model to approximate local planner behaviour while preserving the geometric feasibility information provided by the RRT-generated trajectories.
